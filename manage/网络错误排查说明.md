# 管理后台网络错误排查说明

## 问题描述

前端管理后台请求 `http://localhost:3001/api/admin/auth/login` 时出现网络错误。

## 问题原因

1. **CORS 跨域问题**：
   - 后端 CORS 配置只允许 `http://localhost:3000`
   - 前端管理后台运行在 `http://localhost:3002`
   - 前端 axios 配置使用了完整 URL `http://localhost:3001`，没有走 Vite 代理

2. **请求路径问题**：
   - 前端直接请求 `http://localhost:3001`，绕过了 Vite 代理
   - 应该使用相对路径 `/api`，让 Vite 代理转发请求

## 解决方案

### 1. 后端 CORS 配置（已修复）

已更新 `node/app.js`，支持多个前端源：

```javascript
const allowedOrigins = process.env.CORS_ORIGIN 
  ? process.env.CORS_ORIGIN.split(',').map(origin => origin.trim())
  : ['http://localhost:3000', 'http://localhost:3002'];

app.use(cors({
  origin: function (origin, callback) {
    if (!origin) return callback(null, true);
    if (allowedOrigins.indexOf(origin) !== -1 || process.env.NODE_ENV === 'development') {
      callback(null, true);
    } else {
      callback(new Error('Not allowed by CORS'));
    }
  },
  credentials: true
}));
```

### 2. 前端请求配置（已修复）

已更新 `manage/src/utils/request.js`，开发环境使用代理：

```javascript
// 开发环境使用代理，生产环境使用环境变量配置的地址
const baseURL = import.meta.env.PROD 
  ? (import.meta.env.VITE_API_BASE_URL || 'http://localhost:3001')
  : '/api' // 开发环境使用 Vite 代理
```

### 3. Vite 代理配置（已存在）

`manage/vite.config.js` 中已配置代理：

```javascript
server: {
  port: 3002,
  proxy: {
    '/api': {
      target: 'http://localhost:3001',
      changeOrigin: true
    }
  }
}
```

## 验证步骤

1. **重启后端服务**：
   ```bash
   cd node
   npm start
   ```

2. **重启前端服务**：
   ```bash
   cd manage
   npm run dev
   ```

3. **检查请求**：
   - 打开浏览器开发者工具
   - 查看 Network 标签
   - 请求应该显示为 `http://localhost:3002/api/admin/auth/login`
   - 而不是 `http://localhost:3001/api/admin/auth/login`

4. **测试登录**：
   - 访问 `http://localhost:3002`
   - 尝试登录
   - 应该不再出现网络错误

## 其他可能的问题

### 1. 后端服务未启动

**检查**：
```bash
curl http://localhost:3001/health
```

**解决**：启动后端服务

### 2. 端口冲突

**检查**：
- 后端是否运行在 3001 端口
- 前端是否运行在 3002 端口

**解决**：修改端口配置或关闭占用端口的进程

### 3. 防火墙/安全软件拦截

**检查**：查看浏览器控制台和网络请求详情

**解决**：检查防火墙设置，允许本地端口通信

## 环境变量配置（可选）

如果需要自定义配置，可以创建 `.env` 文件：

**后端 `.env`**：
```env
CORS_ORIGIN=http://localhost:3000,http://localhost:3002
```

**前端 `.env`**：
```env
VITE_API_BASE_URL=http://localhost:3001
```

## 总结

修复后的请求流程：

1. 前端发起请求：`/api/admin/auth/login`
2. Vite 代理转发：`http://localhost:3002/api/admin/auth/login` → `http://localhost:3001/api/admin/auth/login`
3. 后端 CORS 允许：`http://localhost:3002` 在允许列表中
4. 请求成功返回

现在应该可以正常工作了！
