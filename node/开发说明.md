# 开发说明

## 热更新配置

项目已配置 **nodemon** 支持开发时的热更新。

### 启动开发服务器

```bash
cd node
npm run dev
```

### 配置文件

- `nodemon.json` - nodemon 配置文件
- 监听目录：`services`, `controllers`, `routes`, `middleware`, `config`, `utils`
- 监听文件类型：`.js`, `.json`
- 忽略：`node_modules`, `logs`, `uploads`, `*.log`, `*.test.js`

### 工作原理

1. nodemon 会监听指定目录的文件变化
2. 当文件被修改时，自动重启 Node.js 服务
3. 无需手动重启，提高开发效率

### 注意事项

- 修改代码后，nodemon 会自动检测并重启服务
- 重启过程通常需要 1-3 秒
- 如果修改了环境变量（`.env`），需要手动重启服务
- 生产环境请使用 `npm start`（不使用 nodemon）

### 停止服务

按 `Ctrl+C` 停止 nodemon 进程
