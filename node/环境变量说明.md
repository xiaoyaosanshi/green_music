# 环境变量配置说明

## 快速开始

在 `node` 目录下创建 `.env` 文件，复制以下配置并修改相应的值：

```env
# ============================================
# Green Music 后端环境变量配置
# ============================================

# ============================================
# 应用基础配置
# ============================================
# 运行环境: development | production | test
NODE_ENV=development

# 服务器端口
PORT=3001

# CORS 允许的源（前端地址）
CORS_ORIGIN=http://localhost:3000

# ============================================
# MySQL 数据库配置
# ============================================
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password_here
DB_NAME=green_music

# 数据库连接池大小（可选，默认 10）
DB_CONNECTION_LIMIT=10

# ============================================
# Redis 配置（可选，但强烈推荐）
# ============================================
REDIS_HOST=localhost
REDIS_PORT=6379

# Redis 密码（如果设置了密码，取消注释并填写）
# REDIS_PASSWORD=your_redis_password

# ============================================
# JWT 认证配置
# ============================================
# JWT 密钥（必须修改为强随机字符串，建议至少 32 位）
# 生成方式: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
JWT_SECRET=your_jwt_secret_key_change_this_in_production

# JWT 过期时间（可选，单位：秒，默认 7 天）
# JWT_EXPIRES_IN=604800

# ============================================
# 文件上传配置
# ============================================
# 上传文件存储目录
UPLOAD_DIR=./uploads

# 最大文件大小（单位：字节，默认 50MB）
# 50MB = 50 * 1024 * 1024
MAX_FILE_SIZE=52428800

# 允许的音频文件类型（MIME 类型，逗号分隔）
ALLOWED_AUDIO_TYPES=audio/mpeg,audio/mp3,audio/wav,audio/ogg,audio/m4a

# 允许的图片文件类型（MIME 类型，逗号分隔）
ALLOWED_IMAGE_TYPES=image/jpeg,image/png,image/webp,image/gif

# ============================================
# 日志配置
# ============================================
# 日志存储目录
LOG_DIR=./logs

# 日志级别: error | warn | info | debug | verbose
LOG_LEVEL=info

# ============================================
# 对象存储配置（可选，用于生产环境）
# ============================================
# 如果使用 MinIO 或云对象存储（如阿里云 OSS、腾讯云 COS），取消注释并配置

# 存储类型: local | minio | oss | cos
# STORAGE_TYPE=local

# MinIO 配置
# MINIO_ENDPOINT=localhost:9000
# MINIO_ACCESS_KEY=minioadmin
# MINIO_SECRET_KEY=minioadmin
# MINIO_BUCKET=green-music
# MINIO_USE_SSL=false

# 阿里云 OSS 配置
# OSS_REGION=oss-cn-hangzhou
# OSS_ACCESS_KEY_ID=your_access_key_id
# OSS_ACCESS_KEY_SECRET=your_access_key_secret
# OSS_BUCKET=green-music
# OSS_ENDPOINT=https://oss-cn-hangzhou.aliyuncs.com

# 腾讯云 COS 配置
# COS_REGION=ap-guangzhou
# COS_SECRET_ID=your_secret_id
# COS_SECRET_KEY=your_secret_key
# COS_BUCKET=green-music-1234567890

# CDN 域名（如果使用了 CDN，配置后文件 URL 会使用 CDN 地址）
# CDN_DOMAIN=https://cdn.example.com

# ============================================
# 推荐系统配置（可选）
# ============================================
# 推荐结果缓存时间（秒，默认 3600）
# RECOMMEND_CACHE_TTL=3600

# 推荐计算任务队列（如果使用 Bull/BullMQ）
# REDIS_QUEUE_HOST=localhost
# REDIS_QUEUE_PORT=6379

# ============================================
# 安全配置
# ============================================
# 登录失败锁定时间（分钟，默认 30）
# LOGIN_LOCKOUT_MINUTES=30

# 登录失败最大次数（默认 5）
# LOGIN_MAX_ATTEMPTS=5
```

## 必需的环境变量

以下环境变量是**必须配置**的：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `NODE_ENV` | 运行环境 | `development` 或 `production` |
| `PORT` | 服务器端口 | `3001` |
| `DB_HOST` | 数据库主机 | `localhost` |
| `DB_PORT` | 数据库端口 | `3306` |
| `DB_USER` | 数据库用户名 | `root` |
| `DB_PASSWORD` | 数据库密码 | `your_password` |
| `DB_NAME` | 数据库名称 | `green_music` |
| `JWT_SECRET` | JWT 密钥 | **必须修改为强随机字符串** |

## 可选的环境变量

以下环境变量有默认值，可根据需要修改：

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `CORS_ORIGIN` | CORS 允许的源 | `http://localhost:3000` |
| `DB_CONNECTION_LIMIT` | 数据库连接池大小 | `10` |
| `REDIS_HOST` | Redis 主机 | `localhost` |
| `REDIS_PORT` | Redis 端口 | `6379` |
| `REDIS_PASSWORD` | Redis 密码 | 无（可选） |
| `UPLOAD_DIR` | 上传目录 | `./uploads` |
| `MAX_FILE_SIZE` | 最大文件大小（字节） | `52428800` (50MB) |
| `ALLOWED_AUDIO_TYPES` | 允许的音频类型 | `audio/mpeg,audio/mp3,audio/wav` |
| `ALLOWED_IMAGE_TYPES` | 允许的图片类型 | `image/jpeg,image/png,image/webp` |
| `LOG_DIR` | 日志目录 | `./logs` |
| `LOG_LEVEL` | 日志级别 | `info` |

## 生成 JWT_SECRET

**重要**：`JWT_SECRET` 必须是一个强随机字符串，建议使用以下命令生成：

```bash
# 使用 Node.js 生成 32 字节的随机十六进制字符串
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# 或者使用 OpenSSL
openssl rand -hex 32
```

## 环境变量使用位置

- **`app.js`**: `NODE_ENV`, `CORS_ORIGIN`
- **`server.js`**: `NODE_ENV`, `PORT`
- **`config/database.js`**: `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_CONNECTION_LIMIT`
- **`config/redis.js`**: `REDIS_HOST`, `REDIS_PORT`, `REDIS_PASSWORD`
- **`middleware/auth.js`**: `JWT_SECRET`
- **`middleware/upload.js`**: `UPLOAD_DIR`, `MAX_FILE_SIZE`, `ALLOWED_AUDIO_TYPES`, `ALLOWED_IMAGE_TYPES`
- **`utils/logger.js`**: `LOG_DIR`, `LOG_LEVEL`

## 注意事项

1. **生产环境**：必须修改 `JWT_SECRET` 为强随机字符串
2. **数据库密码**：不要使用弱密码，建议使用强密码
3. **Redis**：虽然可选，但强烈推荐使用 Redis 以提升性能
4. **文件上传**：生产环境建议使用对象存储（MinIO/OSS/COS）而不是本地存储
5. **CORS**：生产环境应配置为实际的前端域名，不要使用 `*`
