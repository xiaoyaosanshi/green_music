# 管理后台接口实现说明

## 概述

根据后台管理系统设计文档，已实现所有管理后台相关的API接口。

## 已实现的接口

### 1. 认证接口
- ✅ `POST /api/admin/auth/login` - 管理员登录
- ✅ `GET /api/admin/auth/info` - 获取管理员信息

### 2. 音乐管理接口
- ✅ `GET /api/admin/songs` - 获取音乐列表
- ✅ `GET /api/admin/songs/:id` - 获取音乐详情
- ✅ `POST /api/admin/songs` - 创建音乐（支持文件上传）
- ✅ `PUT /api/admin/songs/:id` - 更新音乐（支持文件上传）
- ✅ `DELETE /api/admin/songs/:id` - 删除音乐
- ✅ `PUT /api/admin/songs/:id/status` - 上架/下架音乐
- ✅ `POST /api/admin/songs/batch` - 批量操作音乐

### 3. 专辑管理接口
- ✅ `GET /api/admin/albums` - 获取专辑列表
- ✅ `GET /api/admin/albums/:id` - 获取专辑详情
- ✅ `POST /api/admin/albums` - 创建专辑
- ✅ `PUT /api/admin/albums/:id` - 更新专辑
- ✅ `DELETE /api/admin/albums/:id` - 删除专辑

### 4. 歌词管理接口
- ✅ `GET /api/admin/lyrics` - 获取歌词列表
- ✅ `POST /api/admin/lyrics` - 上传官方歌词
- ✅ `PUT /api/admin/lyrics/:id/review` - 审核歌词
- ✅ `DELETE /api/admin/lyrics/:id` - 删除歌词

### 5. 用户管理接口
- ✅ `GET /api/admin/users` - 获取用户列表
- ✅ `GET /api/admin/users/:id` - 获取用户详情
- ✅ `PUT /api/admin/users/:id/status` - 禁用/启用用户
- ✅ `POST /api/admin/users/:id/reset-password` - 重置密码
- ✅ `DELETE /api/admin/users/:id` - 删除用户

### 6. 评论管理接口
- ✅ `GET /api/admin/comments` - 获取评论列表
- ✅ `GET /api/admin/comments/:id` - 获取评论详情
- ✅ `PUT /api/admin/comments/:id/review` - 审核评论
- ✅ `DELETE /api/admin/comments/:id` - 删除评论

### 7. 歌单管理接口
- ✅ `GET /api/admin/playlists` - 获取歌单列表
- ✅ `GET /api/admin/playlists/:id` - 获取歌单详情
- ✅ `PUT /api/admin/playlists/:id` - 更新歌单
- ✅ `DELETE /api/admin/playlists/:id` - 删除歌单

### 8. 数据统计接口
- ✅ `GET /api/admin/statistics` - 获取数据统计概览
- ✅ `GET /api/admin/statistics/users` - 获取用户统计
- ✅ `GET /api/admin/statistics/songs` - 获取音乐统计
- ✅ `GET /api/admin/statistics/playback` - 获取播放统计
- ✅ `GET /api/admin/statistics/recommend` - 获取推荐统计

### 9. 操作日志接口
- ✅ `GET /api/admin/logs` - 获取操作日志列表
- ✅ `GET /api/admin/logs/:id` - 获取日志详情

### 10. 文件上传接口
- ✅ `POST /api/admin/upload` - 上传文件

## 使用说明

### 1. 创建操作日志表

执行以下SQL创建操作日志表：

```bash
mysql -u root -p green_music < node/migrations/create_admin_logs_table.sql
```

或者在MySQL客户端中执行：

```sql
source node/migrations/create_admin_logs_table.sql;
```

### 2. 认证

所有管理后台接口（除登录外）都需要在请求头中携带JWT Token：

```
Authorization: Bearer <token>
```

### 3. 文件上传

音乐和封面图上传使用 `multipart/form-data` 格式：

```javascript
const formData = new FormData();
formData.append('file', audioFile);
formData.append('cover', coverImage);
```

### 4. 批量操作

批量操作接口支持以下操作：
- `delete` - 批量删除
- `online` - 批量上架
- `offline` - 批量下架
- `update_genre` - 批量修改类型

请求示例：
```json
{
  "ids": [1, 2, 3],
  "action": "delete",
  "data": {}
}
```

## 注意事项

1. **操作日志**：操作日志功能已实现，但如果 `admin_logs` 表不存在，会静默失败，不影响功能使用。

2. **文件上传**：上传的文件存储在 `uploads/` 目录下，需要确保该目录有写权限。

3. **权限控制**：所有管理后台接口都通过 `adminAuth` 中间件验证管理员身份。

4. **数据统计**：推荐统计接口目前返回模拟数据，需要根据实际推荐系统实现。

5. **密码重置**：重置密码功能目前只生成重置Token，实际发送邮件功能需要配置邮件服务。

## 测试

可以使用以下方式测试接口：

1. 使用 Postman 或类似工具
2. 使用前端管理后台页面
3. 使用 curl 命令

示例：
```bash
# 管理员登录
curl -X POST http://localhost:3001/api/admin/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"password"}'

# 获取音乐列表（需要Token）
curl -X GET http://localhost:3001/api/admin/songs \
  -H "Authorization: Bearer <token>"
```
