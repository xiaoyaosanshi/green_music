# 管理后台接口测试说明

## 前置条件

### 1. 启动后端服务

```bash
cd node
npm start
```

或使用 nodemon（开发模式）：

```bash
npm run dev
```

### 2. 确保数据库已初始化

执行以下SQL脚本初始化数据库和测试数据：

```bash
# 创建数据库表结构
mysql -u root -p green_music < migrations/init_database.sql

# 创建操作日志表（可选）
mysql -u root -p green_music < migrations/create_admin_logs_table.sql

# 初始化测试数据（包含管理员账号）
mysql -u root -p green_music < migrations/init_all_mock_data.sql
```

或使用 Node.js 脚本初始化：

```bash
node migrations/init_mock_users.js
```

### 3. 默认管理员账号

根据 `init_all_mock_data.sql`，默认管理员账号为：

- **用户名**: `admin`
- **密码**: `Admin123456`
- **角色**: `super_admin`

或

- **用户名**: `operator`
- **密码**: `Operator123456`
- **角色**: `admin`

## 手动测试示例

### 1. 管理员登录

```bash
curl -X POST http://localhost:3001/api/admin/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"Admin123456"}'
```

**响应示例**:
```json
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "admin": {
      "id": 1,
      "username": "admin",
      "email": "admin@greenmusic.com",
      "role": "super_admin"
    }
  }
}
```

### 2. 使用 Token 访问接口

将返回的 token 保存到变量中：

```bash
TOKEN="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

然后使用该 token 访问其他接口：

```bash
# 获取管理员信息
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/admin/auth/info

# 获取数据统计
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/admin/statistics

# 获取音乐列表
curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:3001/api/admin/songs?page=1&pageSize=10"
```

### 3. 测试文件上传

```bash
# 上传文件
curl -X POST http://localhost:3001/api/admin/upload \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/file.jpg"
```

### 4. 测试创建音乐（需要文件）

```bash
curl -X POST http://localhost:3001/api/admin/songs \
  -H "Authorization: Bearer $TOKEN" \
  -F "title=测试歌曲" \
  -F "artist=测试歌手" \
  -F "genre=pop" \
  -F "file=@/path/to/song.mp3" \
  -F "cover=@/path/to/cover.jpg"
```

## 测试结果说明

### 成功响应

所有接口成功时返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {...}
}
```

### 常见错误

1. **401 Unauthorized**: Token 无效或过期
   - 重新登录获取新 Token

2. **403 Forbidden**: 无权限访问
   - 检查是否为管理员账号
   - 检查账号状态是否正常

3. **404 Not Found**: 资源不存在
   - 检查请求的 ID 是否存在

4. **500 Internal Server Error**: 服务器错误
   - 检查后端日志
   - 检查数据库连接

## 注意事项

1. **Token 有效期**: 管理员 Token 有效期为 7 天
2. **文件上传**: 确保上传的文件路径正确，文件大小符合限制
3. **数据库**: 确保数据库中有测试数据，否则某些接口可能返回空列表
4. **端口**: 默认后端服务运行在 3001 端口，如不同请修改脚本中的 BASE_URL

## 完整测试流程

1. 启动后端服务
2. 初始化数据库和测试数据
3. 运行测试脚本或手动测试
4. 检查响应结果
5. 查看后端日志确认操作记录

## 故障排查

### 问题：登录失败

**可能原因**:
- 后端服务未启动
- 数据库未初始化
- 管理员账号不存在
- 密码错误

**解决方法**:
1. 检查后端服务是否运行：`curl http://localhost:3001/health`
2. 检查数据库连接
3. 执行初始化脚本创建管理员账号
4. 确认密码是否正确（注意大小写）

### 问题：接口返回 403

**可能原因**:
- Token 无效
- 不是管理员账号
- 账号被禁用

**解决方法**:
1. 重新登录获取新 Token
2. 确认使用的是管理员账号登录
3. 检查数据库中的账号状态

### 问题：接口返回空数据

**可能原因**:
- 数据库中没有对应数据
- 查询条件不匹配

**解决方法**:
1. 执行初始化脚本添加测试数据
2. 调整查询参数
