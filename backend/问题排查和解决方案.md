# 问题排查和解决方案

## 当前问题

1. **多个服务实例运行**：有多个 Node.js 进程占用 3001 端口
2. **代码已更新但服务未生效**：代码文件已改为 `al.title`，但服务仍在使用旧代码 `al.name`

## 解决方案

### 1. 停止所有 Node.js 进程

**Windows 命令：**
```bash
# 查找占用 3001 端口的进程
netstat -ano | findstr ":3001"

# 停止所有 node.exe 进程
taskkill /F /IM node.exe
```

**或者手动停止：**
- 在任务管理器中找到所有 `node.exe` 进程并结束

### 2. 使用 nodemon 启动开发服务器

```bash
cd node
npm run dev
```

### 3. 验证服务

```bash
# 健康检查
curl http://localhost:3001/health

# 测试歌曲列表接口
curl "http://localhost:3001/api/songs?page=1&pageSize=2"
```

## 已修复的问题

1. ✅ **SQL 字段错误**：`al.name` → `al.title`（albums 表使用 `title` 字段）
2. ✅ **参数类型问题**：所有分页参数已使用 `parseInt()` 确保类型正确
3. ✅ **参数数组问题**：数据库查询方法已处理空参数数组的情况
4. ✅ **热更新配置**：已配置 nodemon，支持开发时自动重启

## 开发工作流

1. 启动服务：`npm run dev`
2. 修改代码：编辑任何 `.js` 文件
3. 自动重启：nodemon 检测到变化后自动重启
4. 测试接口：使用 curl 或 Postman 测试

## 注意事项

- 如果修改了 `.env` 文件，需要手动重启服务
- 如果端口被占用，先停止所有 Node.js 进程
- 生产环境使用 `npm start`（不使用 nodemon）
