# Green Music 后端项目启动说明

## 前置要求

- Node.js >= 16.0.0
- MySQL >= 8.0
- Redis >= 6.0（可选，但强烈推荐）

## 快速启动

### 1. 安装依赖

```bash
cd node
npm install
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env`：

```bash
cp .env.example .env
```

修改 `.env` 文件中的数据库配置：

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=green_music
```

### 3. 创建数据库

登录 MySQL 并创建数据库：

```bash
mysql -u root -p
```

```sql
CREATE DATABASE green_music CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

### 4. 初始化数据库表结构

执行数据库初始化脚本：

**方式一：使用命令行（推荐）**

```bash
# 在项目根目录执行
mysql -u root -p green_music < node/migrations/init_database.sql
```

**方式二：在 MySQL 中执行**

```bash
mysql -u root -p
```

```sql
USE green_music;
SOURCE node/migrations/init_database.sql;
```

**方式三：使用 MySQL Workbench**

1. 打开 MySQL Workbench
2. 连接到 MySQL 服务器
3. 打开 `node/migrations/init_database.sql` 文件
4. 选择数据库 `green_music`
5. 执行脚本

**验证安装**

```sql
USE green_music;
SHOW TABLES;
```

应该看到 20 个表。详细说明请参考 `migrations/README.md`。

### 5. 初始化 Mock 数据（可选，推荐）

为了便于开发和测试，可以初始化一些测试数据：

**使用完整 SQL 脚本（推荐，一步完成）：**

```bash
mysql -u root -p green_music < node/migrations/init_all_mock_data.sql
```

**或者在 MySQL 中执行：**

```sql
USE green_music;
SOURCE node/migrations/init_all_mock_data.sql;
```

这将创建：
- ✅ 10 位歌手、15 张专辑、25 首歌曲
- ✅ 4 个测试用户（用户名：testuser1/testuser2/testuser3/demo，密码：Test123456/Demo123456）
- ✅ 2 个管理员（用户名：admin/operator，密码：Admin123456/Operator123456）
- ✅ 歌单、评论、评分、播放历史等完整测试数据

**注意**：所有密码已使用 bcrypt 加密，可直接用于登录测试。

详细说明请参考 `migrations/MOCK_DATA_README.md`。

### 6. 启动服务

开发环境（自动重启）：
```bash
npm run dev
```

生产环境：
```bash
npm start
```

服务默认运行在 `http://localhost:3001`

## 项目结构说明

```
node/
├── config/              # 配置文件
│   ├── database.js      # MySQL 连接池配置
│   └── redis.js         # Redis 客户端配置

├── services/            # 服务层（业务逻辑）
├── routes/              # 路由定义
├── middleware/          # 中间件
│   ├── auth.js         # JWT 认证
│   ├── rateLimiter.js  # 限流
│   └── validator.js    # 参数校验
├── utils/               # 工具函数
│   ├── response.js     # 统一响应格式
│   ├── constants.js    # 常量定义
│   └── logger.js       # 日志工具
├── algorithms/         # 推荐算法
├── workers/            # 后台任务（推荐计算等）
├── app.js              # Express 应用入口
└── server.js           # 服务器启动文件
```

## 注意事项

1. **数据库连接**：确保 MySQL 服务已启动，数据库已创建
2. **Redis（可选）**：如果未安装 Redis，服务仍可运行，但缓存功能不可用
3. **日志目录**：首次运行会自动创建 `logs/` 目录
4. **上传目录**：开发环境上传文件保存在 `uploads/` 目录

## 常见问题

### 数据库连接失败

检查 `.env` 中的数据库配置是否正确，确保 MySQL 服务已启动。

### Redis 连接失败

如果未安装 Redis，服务仍可运行，但会输出警告日志。如需使用缓存功能，请安装并启动 Redis。

### 端口被占用

修改 `.env` 中的 `PORT` 配置，或关闭占用 3001 端口的其他服务。

## 开发建议

1. 使用 `npm run dev` 启动开发服务器（自动重启）
2. 使用 Postman 或 Apifox 进行 API 测试
3. 查看 `logs/` 目录下的日志文件排查问题
4. 参考 `后端实现方案.md` 了解详细实现细节
